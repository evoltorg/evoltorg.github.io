The Future If Presenting Data

The future of the Web will revolve around using full CSS web layouts rather than table-based Web design.

The difference is in positioning DIV areas with CSS, rather than using tables to hold the content together,

which should be reserved for presenting data.

Why Table-Based Web Design is Bad Form

Tables are designed to display tabular data; not for creative Web page layouts. However, that doesn't mean

we should get rid of, or ignore, tables completely in our markup. Tables are still a vital part of HTML,

as they retain their usefulness in marking up tabular data such as calendars.

Here are a few reasons that table-based design has become bad form in Web design:

  • In table-based design, a website's content and markup are combined in one document, making page file sizes

    unnecessarily large. This forces site visitors to download presentational table data for every page they visit on a site.

  • Table-based design can make it hard to maintain visual consistency throughout a site.

  • It's much more difficult to design a table-based Web layout that complies with accessibility

    laws and standards for the disabled or those accessing the Web via cell phones and PDAs.

    For example, an audio browser will typically read the contents of a table from left to right and top to bottom.

    If a designer uses tables to layout a site's content in a vertical fashion (with the intention to read content down the first column,

    then to the top of the second column, etc.), the screen reader will be reading content out of order, causing confusion to the listener.

  • Markup language isn't intended to define display elements of a document, despite the fact

    that display and structural elements are oftenintimately connected.

Benefits of Using CSS in Web Design

Semantic Mark-up - It's easier to achieve a semantic mark-up (strongly recommended by w3c) using CSS design.

What this means is that code should be meaningful as well as syntactically correct. In order to have semantic meaning,

your DIVs should be assigned descriptive IDs (eg., Assign <div id="content"> to your main content area,

wrap headings in <h1> tags, subheadings in <h2> tags, paragraphs in <p> tags, etc.

It also means using <strong> tags as opposed to <b> tags for bold text and <em>

tags instead of <i> tags for italicized or emphasized text.)

Essentially, the descriptive IDs allow your markup to convey meaning to the reader.

Less Bandwidth Usage - When using CSS design techniques, a site's actual HTML files are separate from your CSS markup,

with both files being caches separately in the browser. This saves bandwidth, because the CSS file detailing design elements

only has to be downloaded by the visitor once, as opposed to table-based designs, where the full layout must be downloaded

with every page load. By converting a table-based Web design into a CSS design, you can often decrease the page size by 50-60%.

Better SEO - CSS designs are better for your search engine optimization efforts, because CSS allows you to separate the page's

structure from its presentation elements. This improves your code to content ratio (under the concept that the more text versus

code is present in your page, the better your site will rank in search engines because of relevancy factors).

Image Credit: www.RedFlyMarketing.com

The RedFly Marketing homepage demonstrates the SEO benefit of content positioning in a CSS Web layout. If this design were

created in a table-based layout, search engines would read the left column's content (less important) before the main content

area in the center of the page (most important for SEO purposes to determine relevancy of keywords in a page).

This can also aid in displaying more relevant contextual ads when using contextual advertising networks such as

Google Adsense or the Yahoo! Publisher Network, which can lead to increased conversions and income. By minimizing

the amount of code as compared to content, you're also making it easier for search engines to spider your site,

due to the fact that the code is more structured and simplified.

Yet another SEO benefit of CSS design is that a designer has the ability to manipulate the order of data as

it will be presented to search engines. This means that the most important areas of a site (such as the main content block)

can be placed earlier in the site's code than a less important element (such as a sidebar navigational menu).

Easier Design Changes - CSS design makes it much easier to make small changes to design elements sitewide

(such as content fonts or colors), because a single value changed in a single stylesheet can instantly change

the appearance of an element across an entire website, where all pages are referencing the same CSS file.

Even complete site redesigns are much easier, because the complete layout of the site is controlled through one external CSS file.

Better Accessibility - Accessibility is greatly enhanced with the use of CSS. CSS-based layouts allow multiple

Web-enabled devices (such as PDAs, cell phones, and TV-based browsers) to draw on the same source pages, all while

being displayed in entirely different styles. The same is true of "printer-friendly" pages, where using a simple

alternative CSS file will remove images and navigation for a printable display of content. Essentially, one page serves all purposes.

Potential Challenges of CSS Layouts

Browser Compatibility - While the latest browser versions have little problem with displaying CSS-based Web designs,

older browsers such as Netscape Navigator 4.x and Internet Explorer 4.x and earlier can only display very basic CSS commands,

whereas more complex stylesheets can cause display problems.

Difficulty in Switching from Tables to CSS - Using CSS for Web layouts requires a different line of thinking than using tables.

This means that some designers may have trouble adapting from a table-based design method that they've used for years to CSS-based design.

The CSS

body {

}

#wrapper {

text-align: left;

margin: 0px auto; /* centering the page */

padding: 0px;

border:0;

width: 760px; /* width of the page */

}

#header {

margin: 0 0 15px 0;

height: 50px;

}

#left {

float: left;

width: 180px;

}

#right {

float: right;

width: 180px;

}

#content {

float: left;

width: 400px;

}

#footer {

clear: both;
