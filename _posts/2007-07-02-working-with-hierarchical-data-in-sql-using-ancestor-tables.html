This document describe an approach to SQL queries of hierarchical data using ancestor tables. Multiple inheritance are allowed but cycles are not allowed for this approach to work.\n

\nThis approach is most appropriate when recursive SQL is not available and the hierarchies are not too deep.\nWith ancestor tables, inserts and deletes are slower but selects are very fast and there is no need to update the whole tree with each insert.\n

\nWe use this hierarchy of objects as an example to illustrate the problem:\n

\n

\n

\nA common way to model this hierarchy is with a Parent table:\n

\n
ObjectID\nParentID\n
5\n3\n
6\n3\n
6\n4\n
7\n4\n
3\n2\n
4\n2\n
2\n1\n

\n

\nThis table can keep the relationships between the objects but does not allow to search for all the ancestors of an object or all the children of a parent using single SQL queries.\n

\nUsing an ancestor table instead of a parent table has the advantage of flattening the relationships between object and simplifying the queries:\n

\n
ObjectID\nAncestorID\nhops\n
5\n3\n1\n
5\n2\n2\n
5\n1\n3\n
6\n3\n1\n
6\n2\n2\n
6\n1\n3\n
6\n4\n1\n
6\n2\n2\n
6\n1\n3\n
7\n4\n1\n
7\n2\n2\n
7\n1\n3\n
3\n2\n1\n
3\n1\n2\n
4\n2\n1\n
4\n1\n2\n
2\n1\n1\n

\n

\nModeling the same hierarchy requires more records when ancestors are explicitly recorded but it simplifies the queries. For example, to get all the children of object-1:\n

\n\nselect distinct object_id from ancestor_table where ancestor_id = 1;\nobject_id\n=======\n5\n6\n7\n3\n4\n2\n

\nIn order to preserve the hierarchical order, a third column called 'hops' can be added to the table. This is useful when query results need to be in order of proximity of the object to the ancestor. Saving the number of hops information in the table also allows to recontruct the table from scratch by first deleting all the records with hops > 1 and then re-inserting all the ancestors. The records with hops=1 are equivalent to the parent table, hops> 1 are the ancestor records.\n

\nIt is also important to have the ability to add and delete associations from this graph without having to re-write the complete tree to the database. This is the general algorithm for inserting and deleting an association in the graph:\n

\n
To add an association between child1 and parent1, find all the parents of parent1 and attach them to each children of child1
\n

\nSimilarly, for deleting an association:\n

\n
To remove an association between child1 and parent1, find all the parents of parent1 and remove them from each child of child1
\n

\nFor this solution to work, redundant associations need to be recorded in the database. In our example, there is two associations between object-6 and object-2, one using object-3 and one using object-4. They both need to be present in the ancestor table. This allows the delete function to delete only one of the two associations. In our example, even after removing the object-3 to object-2 association, object-6 can still reach object-2.\n

\nThe insert and delete algorithms need to properly maintain the 'hops' value. Updates need to be converted into a delete and insert combination.\n

\nHere are the two algorithms in Perl:\n

\n\nsub insert_association {\n my($child, $parent) = @_;\n # give all group ancestors to all children of group. \n my $rc = sql("select ancestor_id, hops from ancestor_table where object_id = $parent"); \n my $rc2 = sql("select group_id,hops from ancestor_table where ancestor_id = $child "); \n foreach my $obj (@$rc2) { \n foreach my $anc (@$rc) { \n my $hops = $obj->[1] + $anc->[1] + 1; # adjust the hops\n sql("insert into ancestor_table (object_id, ancestor_id, hops) \n values ($obj->[0], $anc->[0], $hops )");\n }\n }\n}

\n\nsub delete_association {\n my($child, $parent) = @_;\n # remove parent and it's ancestors from group and its children \n my $rc = sql("select ancestor_id,hops from ancestor_table where object_id = $parent "); \n my $rc2 = sql("select object_id,hops from ancestor_table where ancestor_id = $child");

foreach my $c (@$rc2) { \n foreach my $p (@$rc) { \n my $hops = $c->[1] + $p->[1] + 1; \n my $rc3 = sql("select oid,hops from ancestor_table \n where object_id = $c->[0] and ancestor_id = $p->[0] and hops = $hops");\n # delete only one association, with the right hops.\n # oid is a postgres generated sequence_id used to identify each association.\n foreach my $o (@$rc3) { \n sql("delete from ancestor_table where oid = $o->[0]"); \n last; \n } \n }\n}\n\n
