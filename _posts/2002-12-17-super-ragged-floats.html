CSS has enabled immense flexibility in the positioning of images on HTML pages. \n If used correctly, it can help create page designs match that of print. Using \n CSS, it is now possible to wrap text tightly around images, similar to that \n seen in printed books. Of course it isn't exactly news that CSS can be used for \n wrapping text. I have seen large number of sites using sliced images for wrapping \n text around ragged outlines. I didn't want that so I present an alternative \n that I just discovered.

Using Image Slices

Some time back, Eric Meyer had put up a neat \n tutorial on creating ragged floats, wrapping text around an irregular outline \n of an image. This, he achieved by slicing the image into horizontal strips and \n placing them one below the other. With my example it would be like this -

To achieve wrapping of text around the ragged outlines of the image, he stacked \n up the image slices and floated them using this class -

\n
  img.flowing { \n      float: left; \n      clear: left; \n      margin: 0 2em 0 0;\n  }
\n

That's it! Pretty simple and clean. But it has its own disadvantages. what \n if I do not want to slice images? What if I want finer control on the wrapping? \n Do I need to re-slice the images with different heights?

\n

An Alternative

\n

In simple words, we create a div with background as this image. Then use spacers \n or ideally divs again to wrap the text around the background image's irregular \n outline. The following is the id definition for the container div - \n

\n
  #toycycle{\n      background-image: url(toycycle.jpg);\n      background-repeat: no-repeat\n      background-attachment: scroll;\n      margin: 0px;\n      padding: 0px;\n  }
\n

This tells the browser to put toycycle.jpg as the background image \n for the div. The scroll value fixes the image relative to the div position \n and allows it to move alongwith the div. Also the no-repeat value prevents \n tiling of the image.

\n

For arranging the outline, you can either use spacers or divs. For both, the \n CSS class remains the same as the earlier method --

\n
  .flowing { \n      float: left; \n      clear: left; \n      margin: 0px 2em 0px 0px;\n  }\n
\n

The float:left attribute allows the image to stick to left side of \n the div and the clear:left attribute clears up any text on the left \n side of the image. You can increase or decrease the space between images and \n text by changing the margin attribute. In case of divs, add a font-size:8px; \n attribute to negate any effects of font size changes on the divs. Now, you use \n the id and class in this way --

\n
<div id="toycycle">\n  <img src="0.gif" alt="0" class="flowing" border="0" width="105" height="20" />\n  <img src="0.gif" alt="0" class="flowing" border="0" width="110" height="20" /> \n  <img src="0.gif" alt="0" class="flowing" border="0" width="115" height="20" /> \n  <img src="0.gif" alt="0" class="flowing" border="0" width="110" height="20" /> \n  <img src="0.gif" alt="0" class="flowing" border="0" width="98"  height="20" />\n...\n  <img src="0.gif" alt="0" class="flowing" border="0" width="220" height="20" />\n  <p> Text goes here </p>\n</div>
\n

In case of divs --

\n
<div id="toycycle">\n  <div class="flowing" style="width:105px; height:20px;">&nbsp;</div>\n  <div class="flowing" style="width:110px; height:20px;">&nbsp;</div>\n  <div class="flowing" style="width:115px; height:20px;">&nbsp;</div>\n  <div class="flowing" style="width:110px; height:20px;">&nbsp;</div>\n  <div class="flowing" style="width:98px; height:20px;" >&nbsp;</div>\n...\n  <div class="flowing" style="width:220px; height:20px;">&nbsp;</div>\n  <p> Text goes here </p>\n</div>
\n

Place \n the transparent spacer gifs before the text and place them only as much required. \n Assuming the background image has a height of 400 pixels, if you take each image \n 20pixels high, you'll need about 20 spacers/divs stacked up to cover up the \n whole image. Adjust the width of each image accordingly. You can experiment \n with height to achieve smooth text flow over the image. I have observed that \n if the height is more or less equal to the text, you get smooth fits.

Why use this method?

\n
    \n
  • It does away with the need of slicing images.
    • \n
  • Easy to change parameters in case the text does not fit well.
    • \n
  • More compatible with 5.x browsers. Eric's method has a problem with IE5.0 \n placing the images above text and Opera 5.x placing text above the images. \n This technique shows up almost similar in most browsers including the latest \n ones.
    • \n
  • It degrades better than Eric's method(if you use divs, which I think are \n preferrable).
    • \n
\n
