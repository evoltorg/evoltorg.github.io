Underline

\nPlease do not use the pair to underline text. Not only is it deprecated, but it may confuse users about whether or not it is a link. If you think something should be underlined, it would probably be more correct to make it bold or italic.\n

Color

\nDo not apply any color to any text via CSS or the tag. The CSS for the site allows any user to customize the colors and fonts, and setting colors would override those settings, possibly hiding your text.\n

The <font> Tag

\nThe tag is not allowed, if you use it we will remove it. All font properties for the site are defined via the CSS and customizable by each user. All markup must be structural; the tag does not denote structure. Also, the tag is deprecated.\n

Script and Code Samples

\nIf you're using block-level code samples (see below), please don't forget to escape all your entities.\n This means, in particular, changing all your > and < characters into &gt; and &lt; entities (respectively) as well as & to &amp; and " to &quote;. You can get a complete list of the accepted character entities from section 24.2 Character entity references for ISO 8859-1 characters of the HTML 4.01 specification at the W3C site.       Writing Style
      Research has shown that the writing on a web site has a significant impact on the usability (and therefore effectiveness) of a site - up to 125% improvement. The following are outline guidelines, which will significantly improve the impact of our copy.
        \n 
      1. Be succinct - no more than 50% of equivalent wordage in print
        \nBECAUSE
        \nReading from screens is substantially slower than from paper. Users intensely dislike reading long texts.
        2. \n 
      2. \n	
          \n	 
        1. Be scannable - structure articles with 2 or 3 headlines (nested if necessary).
          2. \n	 
        2. Use meaningful rather than teaser headlines and subheads (reading a headline should tell users what the section's about).
          3. \n	 
        3. Use text weight and colour to add highlighting and emphasis.
          4. \n	 
        4. Use bulleted lists.
          \n	BECAUSE\n		
            \n		 
          1. Users do not read copy in full (at least not on 1st reading), they scan. Headlines and page titles are often viewed out of context.
            2. \n		 
          2. Users are adept at disregarding everything that does not look like a clear headline and do not waste their time on links which may be a waste of time.
            3. \n		
          \n	
        \n 
        3. \n 
      3. \n	
          \n	 
        1. Structure long documents. Content should not be arbitrarily cut up into page 1, page 2 etc. Each page should encapsulate a discrete topic. Each page should be written as an inverse pyramid, with a clear summary of the page at the start.
          2. \n	 
        2. Secondary and background information should always be available on separate pages accessed from a link.
          3. \n	 
        3. Printable versions should be on a single page.
          \n	BECAUSE
          \n	While users dislike long documents, they particularly dislike having to download several pages to cover one topic. They do like choice over which subtopics to read.
          4. \n	
        \n 
        4.  
      4. Use cool, objective language to build credibility.
        \nBECAUSE
        \nUsers detest marketing copy - a promotional style with boastful claims. Users are busy, they want the facts, and tend to believe content which gives it to them without embellishment.
        5. \n 
      5. \n	
          \n	 
        1. Linked text should describe where the links lead, not give ‘click here’ instructions. An appropriate way to consider linked text is that it should represent the headline of the content users will arrive at.
          2. \n	 
        2. Use link titles to give additional explanation if warranted.
          \n	BECAUSE
          \n	Users do not follow bad links, and links can be presented out of context by certain browsers and search engines. ‘Click here’ instructions also break up the flow of text and make it more difficult to read.
          3. \n	
        \n 
        6. \n
