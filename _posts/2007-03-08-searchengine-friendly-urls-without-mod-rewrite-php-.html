--- layout: post permalink: node/60747 --- First of all excuse me for my a little bit rusty english (i`m from germany) but i hope u will understand the facts. lets make it short and easy ... if u make a website with PHP u know this:

www.domain.com/index.php?thing=thingy

its not really enjoyable for searchengines they like much more statical URLs. k then lets give them what they want! in most cases we will work with mod-rewrite but on one hand its hard to handle and sometimes your provider don´t allow it, so what is to do?

the magic-words are "DirectoryIndex" and "FilesMatch".

in your .htaccess file u write for example:

\nDirectoryIndex eng.php\n

that says the server to use that file as index, so the server is not looking for an index.html anymore. now u need to add:

\nOptions +Multiviews\n\nForceType application/x-httpd-php\n\n

this tell the server to search in the URL for "eng" and handle it as a PHP-File. "eng" is just an example u can use\nany word what u want. so that means if we have the following URL:

www.domain.com/eng/thingy.html (btw looks nice or? :-)

the server don`t search for "thingy.html" in the directory "eng", the server now\nhandle that URL as:

www.domain.com/eng.php...

k thats not bad but how can WE handle that with PHP? Nothing easier like that.\nAdd at least to your .htaccess ...

\nAcceptPathInfo On\n

... and now we are in the game! In our PHP-File (in this example with the name eng.php)\nwe get all informations we need with that:

\n$InfoArray = explode('/', ($_SERVER['REQUEST_URI']));\n

and now we have an array with informations like this:

\n$InfoArray[0] = "eng";\n$InfoArray[1] = "thingy.html";\n

the first we don´t need but the second is interesting, just make a little "replace" like\nthis (the .html doesn`t matter u can name it what ever u want):

\n$InfoArray[1] = str_replace('.html', '', $InfoArray[1]);\n

now we have the information we want ... the name of the page:

\n$InfoArray[1] = "thingy";\n

and we can start working with that. we can make all kind of pages what we want and the\nsearchengines will love it and think its statical. so what it looking better?

this?\nwwww.domain.com/index.php?thing=thingy

or this?\nwww.domain.com/eng/thingy.html

i think the searchengines give u the answer ;-)

thank u for your attantion. have a good time and enjoy your life.
