--- layout: post permalink: node/62048 --- If you are like me and you try to avoid having to maintain one CSS file for IE and another for non-IE, here is a new little trick I figured out that may be helpful in regards to sprites. For this article to make sense you must have a preliminary understanding of CSS sprites, furthermore I've only tested it in IE7 and FF3.\n

\nThe challenge is to have an image in a sprite to get positioned equally on IE and and FireFox let's say. Normally the dirty solution would be to code an exception. The one problem is that the !important is not recognized in some IE7 versions. So while it is easy to trick the browser in IE6 with !important, no such luck in IE7. The other constraint is that we want to minimize maintenance cost, so we want to try to keep only one CSS file.\n

\nLet's examine the problem by example.\n

\n\n .class {\n background: white url('/images/sprite.gif') no-repeat;\n background-position: 0px -82px !important;\n background-position: 0px -79px;\n }\n\n
\nThe above is what we would normally be inclined of doing. We might even want to replace the dimensions to em with same undesired results. Using two CSS files would make the situation even more messy. \n

\nThe solution is quite simple actually: We change the dimensions to a percentage!\n

\nThat's right, we will simply have to change the unit to % and surprisingly it yields same results in both browsers. The only caveat is that positive percentage moves the image up, and negative percentage moves the image down - in other words the opposite to the usual behavior when px or em is used.\n

\nA look at the new code would look something like:\n

\n\n .class {\n background: white url('/images/sprite.gif') no-repeat;\n background-position: 0px 105%;\n }\n\n
\nI hope this will prove helpful as it has helped me.\n
