Formatting numbers for currency display and more.

\n

Formatting numbers so they conform to a specific format can be \n deceivingly tricky. For example, one of the most common tasks is to format a \n number for currency display- an integer followed by two decimals. There are \n more subtitles in such a task than many might think. In this tutorial, I'll \n discuss formatting numbers in JavaScript, such as number rounding, \n formatting a number for currency display, and more.

\n

Number rounding in JavaScript

\n

Lets first talk about number rounding in JavaScript. JavaScript \naccomplishes this with the following method:

\n

Math.round(x)

\n

Using it, any supplied argument is rounded off to the nearest \ninteger, and using the ".5" up rule. For example:

\n 
Math.round(25.9) //returns 26\nMath.round(25.2) //returns 25\nMath.round(-2.58) //returns -3
\n

If purging numbers of decimals is all that you require, class \ndismissed!

\n

Taking things a few decimal further

\n

Formatting numbers to specific decimal points still entails\nMath.round(), but padded with a little \nmultiplication and division. See if you can identify the magic formula involved, \nwith the below examples:

\n
var original=28.453\n1) //round "original" to two decimals\nvar result=Math.round(original*100)/100  //returns 28.45\n2) // round "original" to 1 decimal\nvar result=Math.round(original*10)/10  //returns 28.5\n3) //round 8.111111 to 3 decimals\nvar result=Math.round(8.111111*1000)/1000  //returns 8.111
\n

In case you haven't picked up on it, the formula to round any \nnumber to x decimal points is:

\n

1) Multiple the original number by 10^x (10 to the power of x)
\n2) Apply Math.round() to the result
\n3) Divide result by 10^x

\n

Formatting numbers for currency display

\n

Now, lets move on to formatting numbers for currency display in \n JavaScript. You may be tempted to simply use number rounding to first shift \n the number's decimal places (via multiplication), round it, then shift the \n decimal back (via division) to pound the number into your hard earned \n dollar, though that won't work in many cases. For example, consider the \n number 120. Number rounding certainly won't get you to 120.00.

\n

To easily format numbers for a specific number of trailing decimals or \n total digits (aka padding), JavaScript 1.5 introduces the below two nifty \n methods:

\n \n \n \n \n \n \n \n \n \n \n \n \n \n
MethodsDescription
Number.toFixed(x)Formats any number for "x" number of \n trailing decimals. The number is rounded up, and "0"s are used after the \n decimal point if needed to create the desired decimal length.
Number.toPrecision(x)Formats any number so it is of "x" length. \n Also called significant digits. A decimal point and "0"s are used if \n needed to create the desired length.
\n

Number.toFixed()

\n

The best way to see all the subtleties of \n toFixed() is to see it in action:

\n 
var profits=2489.8237\nprofits.toFixed(3) //returns 2489.824 (round up)\nprofits.toFixed(2) //returns 2489.82\nprofits.toFixed(7) //returns 2489.8237000 (padding)
\n

Displaying any number in currency format can't get any easier!

\n

Number.toPrecision()

\n

To toPrecision() now:

\n 
var anumber=123.45\nanumber.toPrecision(6) //returns 123.450 (padding)\nanumber.toPrecision(4) //returns 123.5 (round up)\nanumber.toPrecision(2) //returns 1.2e+2 (you figure it out!)
\n

toPrecision() is useful if your \n number must be of a certain length.

\n

Browser considerations

\n

Now, as noted, the last two methods above are JavaScript 1.5 methods. What this \n means is that they'll only work in IE5.5+ and NS6+. The issue of legacy \n browsers not performing the desired formatting operation not withstanding, \n how do you ensure that these two methods at least degrade well? Well, by \n using method detection in your code. For example:

\n 
var profits=2489.8237\nif (profits.toFixed) //if browser supports toFixed() method\nprofits.toFixed(2)
\n

For those of you who also need to ensure legacy browsers such as IE5 also \n perform the desired number formatting operation, well, then it's time to \n roll your own function.
