\nTwo releases in the last couple days; these are pretty important.\n

\n

\nMS99-017 summary:\nRAS and RRAS clients save the password despite the setting of the "Save Password" checkbox.\nMicrosoft has published the following Knowledge Base (KB) article on this\nissue:
\n - Microsoft Knowledge Base (KB) article Q230681,
\n RAS Credentials Saved when "Save Password" Option Unchecked,
\n http://support.microsoft.com/support/kb/articles/q230/6/81.asp
\n - Microsoft Knowledge Base (KB) article Q233303,
\n RRAS Credentials Saved when "Save Password" Option Unchecked,
\n http://support.microsoft.com/support/kb/articles/q233/3/03.asp
\n

\n

\nMS99-018 summary:\n1) "Malformed Favories Icon" There is a buffer overrun hole in the favicon.ico handling; arbitrary code could be executed.\n
\n2) "Legacy ActiveX Control" There is an ActiveX control installed with IE3 that is not used in either IE4 or 5, though it is included with them. It could allow access to the client's hard drive.\n

\n

\nMicrosoft has published the following Knowledge Base (KB) article on this\nissue:
\n - Microsoft Knowledge Base (KB) article Q231450,
\n Update Available for the "Malformed Favorites Icon" Issue in\n Internet Explorer 5,
\n http://support.microsoft.com/support/kb/articles/q231/4/50.asp\n - Microsoft Knowledge Base (KB) article Q231452,
\n Update Available for "Legacy ActiveX Control" Issue in Internet\n Explorer 5,
\n http://support.microsoft.com/support/kb/articles/q231/4/52.asp
