Do you manage more than 1 wordpress? Do you install wordpress on every of your sites? If you doing in this way, you will have a lot of databases and wordpress installations. How about the future update? If you have 100 sites, you need to manage 100 updates. There is something you can do to have many websites on 1 installation only.

All you have to do is upload the wordpress files as usual. Don't install your site at the first place. Then change the following following code in your wp-config.php.

\n< ?php\n// ** MySQL settings ** //\ndefine('DB_NAME', ''); // The name of the database\ndefine('DB_USER', ''); // Your MySQL username\ndefine('DB_PASSWORD', ''); // ...and password\ndefine('DB_HOST', 'localhost'); // 99% chance you won't need to change this value

// table prefix, for hosting multiple intalltaions in the same db.\n$table_prefix = ereg_replace('^(www\.)', '', $_SERVER['HTTP_HOST']);\n$table_prefix = ereg_replace('\.', '', $table_prefix);\n$table_prefix = ereg_replace(’\-’, ”, $table_prefix);\n$table_prefix = $table_prefix . '_';

// Change this to localize WordPress. A corresponding MO file for the\n// chosen language must be installed to wp-includes/languages.\n// For example, install de.mo to wp-includes/languages and set WPLANG to 'de'\n// to enable German language support.\ndefine ('WPLANG', '');

/* Stop editing */

define('ABSPATH', dirname(__FILE__).'/');\nrequire_once(ABSPATH.'wp-settings.php');\n?>\n

Every domain (or sub-domain) you have pointing at this WordPress installation will use a different prefix for it’s tables in the database. So easy :-)

If you want to add more domains, just setup the domain in your server, then go to that domain and start the standard installation.
